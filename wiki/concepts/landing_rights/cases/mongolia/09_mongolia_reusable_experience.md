# 蒙古案例可复用经验

## 1. 文件用途

本文件提炼蒙古案例对其他国家卫星落地许可研究可复用的方法、证据组织方式和防误读规则。可复用的是分析框架，不是蒙古许可制度本身。

## 2. 结论摘要

### 2.1 已确认信息

蒙古案例显示，同一项目可能同时受上位许可清单、通信许可法、频率法、卫星技术附件、设备认证、收费规则和投资法约束。仅找到一个名为“卫星网络许可”的页面不足以形成完整落地结论（官方依据包括：《许可法》第 8.1 条第 9.8、9.10 项；《通信法》第 12-14 条；《无线电波法》第 7-12 条；CRC 第 37/2022 号决议附件）。

### 2.2 分析推断

对其他国家最有效的复用方式，是先识别本国法定许可名称，再把业务模型映射到网络、服务、频率、设备、站点、协调、费用和主体资格；不要从巴西或蒙古的文件编号反推目标国家制度。

### 2.3 待确认事项

本文件的建议仍需在不同国家的官方法律、监管机构实务和具体业务模型中验证，不能视为通用法律结论。

## 3. 可复用的证据链方法

### 3.1 从许可清单到专项规则

蒙古的证据链可分为：

1. 《许可法》确认是否存在法定许可及主管机关；
2. 《通信法》《无线电波法》确认主体、期限、材料和程序；
3. CRC 第 37/2022 号决议附件确认卫星专项技术、MSS、NTN、设备、站点和 ITU 要求；
4. CRC 申请页面确认实际材料、表格和提交入口；
5. 收费和投资法律处理成本与股权问题。

其他国家也应建立“许可清单—实施法律—专项规则—申请页面—附件/费用”的证据链。

### 3.2 主页面与附件分开处理

CRC 第 37/2022 号决议主页面只确认决议编号、日期、附件批准和旧决议废止；实质义务位于附件。可复用经验是：法规主文和附件应分别保存、分别生成 source note，并在正式报告中引用真正承载义务的文件。

### 3.3 现行本地语文本优先

《通信法》和《无线电波法》的英文译文未同步反映部分现行蒙古文修订。可复用经验是：法规版本冲突时，优先核对现行官方本地语合并文本，翻译仅作辅助，并明确记录差异。

## 4. 业务模型必须先拆分

蒙古第 37/2022 号决议附件对一般卫星服务、MSS 和 NTN 分别设置第 4.4、4.5、4.10 条。由此形成的通用工作方法是：

| 业务问题 | 应确认的事实 |
| --- | --- |
| 容量批发 | 卫星容量卖给谁、合同主体、是否在本地使用频率 |
| 终端零售 | 谁向用户签约和收费、是否另需服务许可 |
| 网关 | 在境内还是境外、由谁运营、使用何种频率 |
| MSS/漫游 | 地球站位置、终端频率、当地持牌责任 |
| NTN/D2D | 地面运营商、卫星合作方、频率许可、登记和客户关系 |
| VSAT/固定终端 | 站点数量、设备型号、批量许可或登记机制 |

只有在业务模型清楚后，才能判断许可组合。

## 5. 四项防误读规则

### 5.1 本地主体条款不等于子公司结论

《无线电波法》第 7.1 条写的是依蒙古法律设立并运营的法人或公民。可复用经验是：遇到类似表述，应继续核对分支机构、法人资格、本地代表和合作持牌模式，不得直接改写为“必须设立本地子公司”。

### 5.2 专门条款不等于完全豁免

MSS 第 4.5 条明确频率许可和境外地球站，但没有明文完全排除第 9.10 项许可。可复用经验是：当专项条款只写出某项许可时，应检查它是完整路径、附加要求还是例外，而不是自动解释为其他许可全部豁免。

### 5.3 NTN 登记不等于卫星运营商零售权

第 4.10 条的主体是地面网络服务经营者。可复用经验是：D2D/NTN 规则必须逐字识别谁登记、谁持频率、谁向用户服务，不能把网络合作能力等同于外国卫星运营商的独立零售权。

### 5.4 法定节点不等于项目总周期

蒙古法同时存在材料检查、实质核验、公告、遴选、签约、付款、发证和国际协调节点。可复用经验是：只报告各节点及其起算条件，不把一般期限机械相加为完整审批周期。

## 6. 费用分析方法

蒙古频率收费依赖频段、带宽、功率、业务、覆盖、站点和授予方式；设备认证费又依评估方案。可复用经验是：

1. 先获取计算规则；
2. 再确认现行费率或系数；
3. 建立项目参数表；
4. 核对豁免触发条件和一次性权利费；
5. 最后取得监管机构书面测算或缴费通知。

没有参数和费率时，应输出“计费原则 + 缺失参数”，不得编造金额。

## 7. 站点和设备的接口分析

蒙古规则把频率申请、设备型号合格、站址土地、地方证明和 ITU 登记连接在一起。可复用经验是：站点许可研究不能只搜索“earth station licence”名称，还应检查：

1. 频率申请是否按站点提交；
2. 设备是否按型号认证；
3. 是否需要土地、建设、地方或环境批准；
4. 是否需要投运前测量或验收；
5. 网关、TT&C、VSAT、ESIM 和用户终端是否分类不同；
6. 是否存在批量或系统级许可。

## 8. 外资条款的适用对象必须精确

蒙古《投资法》的 33% 门槛只适用于符合“外国国有法人”定义的投资者。可复用经验是：任何外资比例结论都必须同时记录投资者类型、持股阈值、目标行业、审批机关和是否存在其他法律；不得把特殊投资者规则扩大为所有外资的统一上限。

## 9. 推荐输出结构

其他国家正式案例可继续使用 `00-09` 结构，但每个文件至少应包含：

1. 已确认信息及条文/官方来源；
2. 分析推断；
3. 待确认事项；
4. 业务和主体适用边界；
5. 费用、期限和版本风险；
6. 下一步书面确认清单；
7. Obsidian 相关文件链接。

## 10. 蒙古仍需人工复核的示范清单

1. 蒙古文关键条款的法律翻译；
2. 分支机构是否满足频率许可主体资格；
3. MSS 第 4.5 条与第 9.10 项许可的关系；
4. NTN 第 4.10 条的卫星合作、零售和频率责任；
5. 第 9.10 项下一轮遴选公告和实际流程；
6. CRC 附件表格、费率和设备认证规范；
7. 各业务的项目总周期和实际审批顺序。

## 11. 与巴西案例的关系

巴西案例提供许可、服务、频率、设备、站点、费用和法规的文件结构；蒙古案例进一步说明为何结构可以复用、制度不能复用。对新国家，应同时读取目标国家官方来源、common SOP 和巴西可复用经验，但最终结论只能由目标国家证据链支持。
