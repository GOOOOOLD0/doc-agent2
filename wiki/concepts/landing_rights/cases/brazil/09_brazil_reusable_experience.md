# 巴西案例可复用经验

## 1. 文件用途

本文件用于提炼巴西落地许可案例中可复用的分析经验。

后续分析泰国、蒙古、印尼、墨西哥等国家时，应将本文件作为检查清单和对比基准，但不得直接假设目标国家的许可体系与巴西相同。

## 2. 可复用分析框架

巴西案例可抽象出以下通用分析模块：

1. 监管机构识别；
2. 外国卫星落地权或外国卫星准入；
3. 本地公司或本地代表；
4. 面向终端用户的服务授权；
5. 频率使用和频率协调；
6. 设备认证；
7. 地面站、网关站或空间站许可；
8. 费用；
9. 法规依据；
10. 待确认问题。

## 3. 监管机构识别

分析任何国家时，首先应确认该国负责以下事项的监管机构：

1. 电信服务；
2. 卫星服务；
3. 无线电频谱；
4. 设备认证；
5. 地面站或网关站许可。

在巴西案例中，核心监管机构为 Anatel。

## 4. 外国卫星准入

需要确认目标国家是否存在类似巴西 Foreign Satellite Exploitation Rights 的制度。

不同国家可能使用不同名称，例如：

1. Landing Rights；
2. Market Access；
3. Foreign Satellite Authorization；
4. Space Segment Authorization；
5. Right to Exploit Foreign Satellite。

## 5. 本地主体要求

巴西案例中涉及本地公司 SBH 和 SBS。

分析其他国家时，需要确认：

1. 是否必须设立本地公司；
2. 是否允许外国公司直接申请；
3. 是否需要本地代表；
4. 是否需要本地合作伙伴；
5. 是否存在外资持股比例限制。

## 6. 服务授权

巴西案例中，外国卫星开发权和服务授权是两类不同授权。

分析其他国家时，需要确认：

1. 销售卫星容量是否需要单独许可；
2. 向终端用户提供服务是否需要服务授权；
3. 宽带、移动卫星、VSAT、D2D 是否对应不同许可；
4. 本地服务提供方是否必须持牌。

## 7. 频率协调

巴西案例强调频率协调努力证明。

分析其他国家时，需要确认：

1. 是否需要频率使用许可；
2. 是否需要提交 ITU filing；
3. 是否需要与已有运营商协调；
4. 是否需要提交协调协议；
5. 是否接受协调努力证明；
6. 未完成协调时是否只能获得次级授权；
7. 是否需要干扰分析报告。

## 8. 设备认证

巴西案例中，网络设备和用户设备需要认证。

分析其他国家时，需要确认：

1. 用户终端是否需要 type approval；
2. 网关设备是否需要认证；
3. 是否要求本地实验室测试；
4. 是否接受国际认证；
5. 是否需要设备进口许可或标签要求。

## 9. 站点许可

巴西案例中，地面站、网关站和空间站需要许可，NGSO 系统可按系统许可。

分析其他国家时，需要确认：

1. 网关站是否需要许可；
2. 地球站是否需要许可；
3. TT&C 站是否需要许可；
4. 用户终端是否需要单独许可；
5. NGSO 系统是否允许批量许可；
6. 商业运营是否必须等待全部站点许可完成。

## 10. 费用

巴西案例中涉及外国卫星开发权费用、服务授权费用、TFI 和 TFF。

分析其他国家时，需要确认：

1. 申请费；
2. 频率使用费；
3. 站点许可费；
4. 设备认证费；
5. 年费；
6. 续期费；
7. 是否按系统、按站点、按终端或按频段收费。

## 11. 法规依据

巴西案例中包括上位电信法、卫星开发规则、技术运行要求、设备认证规则、站点许可规则和费用规则。

分析其他国家时，也应建立法规依据清单，包括：

1. 上位法律；
2. 卫星服务规则；
3. 频率管理规则；
4. 服务授权规则；
5. 设备认证规则；
6. 站点许可规则；
7. 费用规则；
8. 官方申请指南。

## 12. 不能直接照搬的内容

以下内容不能直接套用到其他国家：

1. 巴西的许可名称；
2. 巴西的申请主体设计；
3. 巴西的费用金额；
4. 巴西的监管机构名称；
5. 巴西的具体申请流程；
6. 巴西的法规编号；
7. 巴西的 primary / secondary 授权安排。

## 13. 正确使用方式

分析其他国家时，应将巴西案例作为：

1. 检查清单；
2. 对比基准；
3. 风险提示；
4. 报告结构样板。

不得假设其他国家一定具有与巴西相同的许可体系。

## 14. 推荐 Agent 工作方式

Agent 在分析新国家时，应执行以下步骤：

1. 读取 common 文件夹中的国家落地许可分析 SOP；
2. 读取巴西案例索引；
3. 读取本文件中的可复用经验；
4. 对目标国家进行开放式检索；
5. 识别目标国家自己的许可体系；
6. 将目标国家许可映射到通用分析模块；
7. 与巴西流程进行对比；
8. 标记待确认问题；
9. 输出标准化报告。
