# 卫星落地许可来源清单模板

## 1. 文件用途

source_inventory 用于记录某个国家卫星落地许可分析中找到的资料来源。

它不是正式分析报告，只用于回答三个问题：

1. 找到了哪些来源；
2. 哪些来源需要人工下载或人工复制；
3. 后续哪些来源可用于生成 cases 文件。

---

## 2. 保存路径

国家来源清单应保存到：

`wiki/raw/articles/landing_rights/<country>/source_inventory.md`

例如：

`wiki/raw/articles/landing_rights/thailand/source_inventory.md`

---

## 3. 来源清单表格

source_inventory.md 必须只使用以下表格格式，不得自行增加复杂表头：

| 序号 | 来源名称 | URL | 对应用途 | 获取状态 | 本地保存路径 | 备注 |
|---|---|---|---|---|---|---|

字段说明：

- 来源名称：网页、PDF、法规、通知、指南、申请表或官网入口名称；
- URL：来源链接；
- 对应用途：监管机构、落地权、服务授权、频率协调、设备认证、站点许可、费用、法规依据等；
- 获取状态：已下载 / 已保存网页正文 / 需人工获取 / 待确认；
- 本地保存路径：如果 Agent 已保存原始资料，应填写 raw/ 下的本地路径；
- 备注：简短说明访问情况或下一步动作。

---

## 4. 原始资料自动保存规则

如果 Agent 能够直接获取来源内容，应自动保存原始资料。

保存规则：

1. PDF、Word、官方通知、申请指南保存到：

   `wiki/raw/papers/landing_rights/<country>/`

2. 网页正文、政府服务页面、法规页面、公告页面保存到：

   `wiki/raw/articles/landing_rights/<country>/`

3. 截图、流程图、网页截图保存到：

   `wiki/raw/assets/landing_rights/<country>/`

保存后，Agent 必须在 source_inventory.md 的“本地保存路径”中填写对应路径。

如果 Agent 无法访问网页、无法下载 PDF、无法提取正文，或遇到验证码、反爬、403、404、跳转异常等情况，不得忽略该来源，应在“获取状态”中写：

`需人工获取`

备注中写明：

`需要用户人工下载 PDF 或复制网页正文。`

Agent 不得因为无法解析网页，就生成空的 cases 文件。

## 5. 填写规则

Agent 应优先记录具体来源，而不是只记录官网首页。

优先级如下：

1. 具体法规；
2. 官方通知；
3. PDF 文件；
4. 申请指南；
5. 费用表；
6. 政府服务页面；
7. 官网入口页。

如果暂时只能找到官网首页，也可以记录，但备注中必须写：

`入口页，需继续定位具体法规或通知。`

如果网页无法访问或正文无法提取，不得忽略该来源，应记录 URL，并在“是否需要人工获取”中写：

`是`

备注中写：

`需要用户人工下载 PDF 或复制网页正文。`

---

## 6. 禁止事项

Agent 不得执行以下行为：

1. 不得生成复杂表头；
2. 不得加入长段解释；
3. 不得使用“待填充链接”；
4. 不得编造法规名称或 URL；
5. 不得因为网页无法抓取就忽略来源；
6. 不得在 source_inventory 尚未完成前生成空的 01-09 cases 文件。

---

## 7. 输出要求

生成 source_inventory.md 后，Agent 只需简短说明：

1. 已找到多少个来源；
2. 其中哪些需要人工获取；
3. 下一步建议优先处理哪些来源。
