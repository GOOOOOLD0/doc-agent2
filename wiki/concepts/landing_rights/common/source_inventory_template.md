# 卫星落地许可来源清单模板

## 1. 文件用途

本文件规定 Agent 在分析某个国家卫星落地许可前，如何整理资料来源清单。

source_inventory 的作用不是生成最终结论，而是先记录：

1. 找到了哪些来源；
2. 来源是否官方；
3. 来源对应哪个分析模块；
4. Agent 是否能访问和提取正文；
5. 是否需要用户人工下载或复制内容。

---

## 2. 保存路径

国家来源清单应保存到：

`wiki/raw/articles/landing_rights/<country>/source_inventory.md`

例如：

`wiki/raw/articles/landing_rights/thailand/source_inventory.md`

如果目录不存在，Agent 应先创建目录。

---

## 3. 来源清单表格

source_inventory.md 必须使用以下表格格式：

| 序号 | 来源名称 | URL | 发布机构 | 来源类型 | 是否官方 | 对应模块 | 可访问 | 可提取正文 | 需人工处理 | 备注 |
| -- | ---- | --- | ---- | ---- | ---- | ---- | --- | ----- | ----- | -- |

字段说明：

* 来源名称：网页、PDF、法规、通知、指南或申请表名称；
* URL：来源链接；
* 发布机构：如 NBTC、Thailand.go.th、Royal Gazette、MDES、ITU；
* 来源类型：官网页面、政府门户、官方通知、法规、PDF、申请指南、非官方文章等；
* 是否官方：是 / 否 / 待确认；
* 对应模块：监管机构、外国卫星落地权、服务授权、频率协调、设备认证、站点许可、费用、法规依据；
* 可访问：可访问 / 无法访问 / 部分；
* 可提取正文：可提取正文 / 无法提取 / 部分；
* 需人工处理：需人工 / 不需人工；
* 备注：说明用途、限制或下一步处理方式。

---

## 4. 无法访问来源的处理规则

如果 Agent 无法访问网页或 PDF，不得忽略该来源。

应在表格中记录该来源，并标注：

* 可访问：否；
* 可提取正文：否；
* 需人工处理：是；
* 备注：需要用户人工下载 PDF 或复制网页正文。

Agent 不得因为无法解析网页，就生成空的 cases 文件。

---

## 5. 来源优先级

Agent 应优先记录：

1. 通信监管机构官网；
2. 政府门户网站；
3. 官方法律法规；
4. 官方通知、公告、指南和申请表；
5. 官方费用表；
6. ITU 或区域组织资料；
7. 律所、咨询机构或行业报告；
8. 新闻或博客。

关键结论应优先依赖官方来源。非官方来源只能作为补充参考。

---

## 6. 输出要求

生成 source_inventory.md 后，Agent 应说明：

1. 哪些来源可以直接用于生成 cases 文件；
2. 哪些来源需要人工下载；
3. 哪些来源需要人工复制正文；
4. 哪些模块资料仍然不足；
5. 下一步建议先处理哪些来源。

在 source_inventory.md 尚未完成前，Agent 不应生成空的 01-09 cases 文件。
