# 国家卫星落地许可案例规范

## 1. 用途与职责边界

本文件是国家卫星落地许可 `00-09` 案例的唯一结构与质量规范，用于约束：

- 案例存放位置、编号、文件名和 frontmatter；
- 各编号文件的固定用途和必须覆盖的信息；
- 已确认信息、分析推断和待确认事项的表达；
- Source Note 引用、证据边界、版本和翻译风险；
- `10_` 以后国家特殊专题文件；
- 自动生成案例的最低验收要求。

执行流程由：

```text
skills/satellite-landing-rights/SKILL.md
```

统一维护。

来源台账、Source Note 和来源优先级分别由以下文件维护：

```text
source_inventory_template.md
source_note_template.md
source_priority_rules.md
```

国家事实只能保存在对应国家的 `raw/` 和 `cases/` 目录中，不得写入本文件。

---

## 2. 路径、命名与 frontmatter

### 2.1 固定路径

国家原始资料：

```text
wiki/raw/landing_rights/<country>/
```

国家正式案例：

```text
wiki/concepts/landing_rights/cases/<country>/
```

通用规则：

```text
wiki/concepts/landing_rights/common/
```

`<country>` 使用英文小写和下划线，例如：

```text
mongolia
south_africa
united_arab_emirates
```

不得将国家案例写入：

```text
wiki/concepts/landing_rights/common/
```

### 2.2 标准文件

每个国家标准案例由十个独立文件组成：

```text
00_<country>_case_index.md
01_<country>_landing_overview.md
02_<country>_foreign_satellite_rights.md
03_<country>_service_authorization.md
04_<country>_frequency_coordination.md
05_<country>_equipment_certification.md
06_<country>_station_licensing.md
07_<country>_fee_list.md
08_<country>_regulations.md
09_<country>_reusable_experience.md
```

不得：

- 合并为 `00-09.md`；
- 改变编号及用途；
- 使用另一套国家专用编号替代标准文件；
- 为填满十个文件而编造内容。

### 2.3 通用 frontmatter

自动生成文件统一使用：

```yaml
---
country: <Country English Name>
topic: <topic>
case_type: structured_case
language: zh-CN
review_status: draft
last_reviewed: YYYY-MM-DD
---
```

可选字段：

```yaml
source_document: <source set or inventory id>
supersedes: <old file path>
```

自动生成内容不得标记为：

```text
human_reviewed
approved
final
```

## 3. 通用正文与信息状态

### 3.1 通用正文

除 `00` 和 `09` 外，优先使用：

```markdown
# <国家><主题标题>

## 1. 文件用途

## 2. 结论摘要

### 2.1 已确认信息

### 2.2 分析推断

### 2.3 待确认事项

## 3. 主管机构与法律依据

## 4. 适用对象与触发条件

## 5. 主要要求或程序

## 6. 费用、期限与有效期

## 7. 证据边界与版本风险

## 8. 官方来源记录

## 9. 相关文件
```

不适用的章节可删除，不得保留只有标题的空节。

### 3.2 信息状态

**已确认信息**

有现行法律、法规、正式决议、政府公报、监管机构正式页面、指南、表格或公告直接支持。

**分析推断**

根据多个来源综合得出，但官方资料没有直接给出完整结论。必须说明推断依据及不确定性。

**待确认事项**

适用于以下情况：

- 官方资料未公开或无法访问；
- 附件未获取；
- 不同语言版本、网页或文件存在冲突；
- 适用主体、生效状态、费用、期限或流程不明确。

统一写法：

> 未在现有官方资料中确认。

## 4. 证据与适用边界

来源等级和使用规则以：

```text
[[source_priority_rules|落地许可资料来源优先级规则]]
```

为准。

关键结论必须能够追溯到：

```text
wiki/raw/landing_rights/<country>/source_notes/<source_id>.md
```

推荐引用方式：

```markdown
来源：[[<source_id>]]
```

或：

```markdown
依据 Source Note：`<source_id>`
```

每项重要结论应说明：

- 支持该结论的来源；
- 关键条款或原文位置；
- 能够支持的范围；
- 不能支持的范围；
- 版本、翻译、附件或修订风险。

不得：

- 将一般许可程序写成卫星项目完整审批周期；
- 将多个阶段时限机械相加为总周期；
- 将收费规则或公式写成具体项目金额；
- 将法规主页代替法规附件；
- 将申请页面说明扩大为法律义务；
- 将特定主体限制扩大到所有外国主体；
- 将一种许可的期限套用到其他许可；
- 静默忽略不同来源之间的冲突；
- 使用其他国家案例填补目标国家证据缺口。

巴西等已有案例只能用于结构参考、检查维度和跨国比较，不能作为目标国家的法律依据。

## 5. `00-09` 文件规范

| 编号 | 文件用途 | 必须覆盖的信息 |
| --- | --- | --- |
| `00` | 国家案例索引 | 案例状态、核心结论、`01-09` 文件目录、主要来源、关键证据缺口、原始资料入口 |
| `01` | 落地许可总览 | 主管机构、整体许可链、服务/频率/设备/站点关系、外国主体进入路径、主要风险 |
| `02` | 外国卫星准入 | 外国卫星或境外运营商是否可进入、本地实体或合作方、外资和控制权限制、系统认可或特殊批准 |
| `03` | 服务授权 | 许可名称、适用业务、申请主体、材料、程序、有效期、续期、暂停撤销、与频率许可关系 |
| `04` | 频率许可与协调 | 分配/指配/使用许可、频段和业务、申请材料、国内审查、ITU及跨境协调、干扰保护、期限续期 |
| `05` | 设备认证 | 适用设备、认证类型、测试和材料、进口与标签、费用周期、有效期、豁免或简化程序 |
| `06` | 地面站与网关许可 | 站点类型、主管机构、位置土地、设备天线、频率协调、建设验收、共享或境外网关、临时站和移动站 |
| `07` | 费用清单 | 申请费、许可费、频率费、监管费、认证费、收费机构、计算方法、币种、缴费时间、费率版本 |
| `08` | 法规依据 | 核心法律、法规、决议、收费规则、申请页面、各来源作用和使用边界、法规层级、版本与翻译风险 |
| `09` | 可复用经验 | 有效研究方法、官方入口、容易混淆的许可边界、版本和附件风险、可复用检查项、不可迁移的国家特殊规则 |

### 模块边界

| 模块 | 含义 |
| --- | --- |
| 外国卫星准入 | 外国卫星系统、容量或境外运营主体进入目标市场 |
| 服务授权 | 建设、运营网络或向用户提供卫星通信服务 |
| 频率许可 | 使用频谱以及进行国内、跨境和 ITU 协调 |
| 设备认证 | 终端和其他无线电设备的合规、测试和进口 |
| 地面站许可 | 网关、地球站、关口站和相关无线电台站 |

不得将不同模块的许可合并成一个不存在的统一许可证。

## 6. 特殊文件要求

### 6.1 `00` 索引

`00_<country>_case_index.md` 应最后生成或更新，并至少包含：

```markdown
# <国家>卫星落地许可案例索引

## 1. 案例状态

## 2. 核心结论

## 3. 文件目录

| 编号 | 文件 | 状态 | 说明 |
| --- | --- | --- | --- |

## 4. 主要官方来源

## 5. 关键待确认事项

## 6. 原始资料与规范
```

所有链接必须指向真实存在的文件。

### 6.2 `08` 法规依据

`08_<country>_regulations.md` 应重点体现来源的证据作用，而不是只罗列法规名称。

推荐结构：

```markdown
# <国家>法规依据

## 1. 文件用途

## 2. 结论摘要

### 2.1 已确认信息

### 2.2 分析推断

### 2.3 待确认事项

## 3. 核心法律和规则

| 来源 | 关键作用 | 主要条款或内容 | 使用边界 |
| --- | --- | --- | --- |

## 4. 监管机构申请和说明页面

| 来源 | 主要用途 | 当前证据状态 |
| --- | --- | --- |

## 5. 法规层级和主题映射

## 6. 版本、翻译和修订风险

## 7. 仍需补充的官方文件

## 8. 相关文件
```

### 6.3 `09` 可复用经验

`09_<country>_reusable_experience.md` 中的内容必须明确属于：

- 研究经验；
- 分析框架；
- 检查清单；
- 国家特殊规则。

不得将目标国家经验写成其他国家的法律事实。

推荐结构：

```markdown
# <国家>案例可复用经验

## 1. 可复用的研究方法

## 2. 有效的官方来源入口

## 3. 容易混淆的许可边界

## 4. 版本、翻译和附件风险

## 5. 可加入通用检查清单的事项

## 6. 不可直接迁移的国家特殊规则
```

### 6.4 `10_` 以后扩展文件

仅当内容无法合理归入 `00-09` 时，才允许增加：

```text
10_<country>_<special_topic>.md
11_<country>_<special_topic>.md
```

适用情形包括：

- 特殊国家安全审查；
- 独立卫星系统登记；
- 特殊招标或名额制度；
- NTN、D2D 或 direct-to-device 专项制度；
- 特殊进口、数据本地化或网络安全要求；
- 国家特有操作指南。

扩展文件不得用于规避标准 `00-09`。

## 7. 内部链接规范

使用：

```markdown
[[文件名|显示名称]]
```

要求：

- 只链接真实存在的文件；
- 使用明确的中文显示名称；
- 删除或合并旧规范文件前先更新引用；
- 国家案例优先链接本国 `00-09`、本国 Source Notes、`case_spec` 和 `source_priority_rules`。

## 8. 生成后校验清单

生成或更新案例后，至少检查：

1. 文件位于 `wiki/concepts/landing_rights/cases/<country>/`；
2. 文件名和编号用途正确；
3. frontmatter 完整，`review_status: draft`；
4. 关键结论可追溯到具体 Source Note 或官方来源；
5. 已确认、分析推断和待确认事项明确区分；
6. 费用、期限、主体资格和许可名称没有无来源补充；
7. 规则的适用范围和使用边界得到保留；
8. 冲突、版本、翻译和附件风险没有被静默省略；
9. 不存在空文件、重复文件和无效内部链接；
10. `00` 索引与实际文件一致；
11. 巴西等参考案例没有被用作目标国家事实依据；
12. 法律名称、条款归属和生效状态仍需人工复核的内容已明确列出。

未通过校验时，不得将案例视为完成版本。
